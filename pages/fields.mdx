## What are Fields

The `fields` array within each collection defines all the inputs that will be used when editing or creating articles, along with any validators that should be applied to them.

```json
 "collections": {
    "page": {
      "title": "Page",
      "files": "/pages/**/*.mdx",
      "fields": [
        {
          "name": "title",
          "label": "Title",
          "validators": [
            "required"
          ]
        },
        {
          "name": "content",
          "label": "Body",
          "type": "richtext",
          "validators": [
            "required"
          ]
        }
      ]
    }
  }
```

Each `field` can have a specified `type` that determines the type of input widget to be used. If no type is specified, the default is a text input widget.

The [AdminEdit](/admin-views#admin-edit) component uses the information provided in the fields array to render the appropriate input widgets for each field.

## Adding Widgets

Widgets can be added through the [SlimplateProvider](http://localhost:3000/admin-views#admin-project), by default we offer a few [widgets](/fields#provided-default-widgets-types) which you can import.

```jsx
import {widgets } from '@slimplate/react-flowbite-github'

 <SlimplateProvider
    widgets={widgets}
    corsProxy={window?.slimplate?.corsProxy}
    backendURL={window?.slimplate?.backendURL}
    monoProject={window?.slimplate?.project}
  >
    <App />
  </SlimplateProvider>
```

## Default Widgets Types

Here is a list of types we offer by default:

*   color
*   date
*   string
*   number
*   richtext
